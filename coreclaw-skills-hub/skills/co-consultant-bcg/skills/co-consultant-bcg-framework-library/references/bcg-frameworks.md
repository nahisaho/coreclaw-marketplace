# BCG Frameworks — Detailed Reference

Read this reference when applying BCG-specific frameworks.

## BCG Growth-Share Matrix

### Structure (2×2)
| | High Market Growth | Low Market Growth |
|---|---|---|
| **High Relative Share** | ⭐ Star: Invest to maintain | 💰 Cash Cow: Harvest profits |
| **Low Relative Share** | ❓ Question Mark: Invest or divest | 🐕 Dog: Divest |

### Application Rules
1. "Relative market share" = your share ÷ largest competitor's share (not absolute).
2. "Market growth" threshold: typically industry average growth rate.
3. Strategy: use Cash Cow profits to fund Stars and promising Question Marks.
4. Dogs are not always "divest" — may serve strategic purpose (blocking competitor).

## BCG Experience Curve

### Principle
- Unit cost decreases by 20-30% each time cumulative production doubles.
- Sources: learning, scale, technology, design improvements.

### Application Rules
1. Plot cumulative volume (x-axis, log) vs unit cost (y-axis, log) — should be linear.
2. If not linear, identify disruptions (technology shifts, regulatory changes).
3. Strategic implication: market share leader has lowest cost position.
4. Experience curve flattens in mature industries — look for discontinuities.

## BCG Strategy Palette

### 5 Strategy Archetypes
| Environment | Archetype | Key Principle |
|------------|-----------|--------------|
| Predictable + Cannot change | **Classical** | Analyze → Plan → Execute at scale |
| Predictable + Can change | **Visionary** | Envision → Build → Persist |
| Unpredictable + Cannot change | **Adaptive** | Experiment → Select → Scale fast |
| Unpredictable + Can change | **Shaping** | Engage → Orchestrate → Evolve |
| Harsh (survival) | **Renewal** | React → Economize → Grow |

### Application Rules
1. Assess environment on 2 dimensions: predictability × malleability.
2. Different business units may need different archetypes.
3. Re-assess archetype as environment changes — strategy type is not permanent.
4. Mismatched archetype (e.g., Classical in unpredictable market) is a common failure mode.

## Hypothesis-Driven Approach

### Structure
1. **Day 1 Answer**: "If I had to answer today, I would say..."
2. **Supporting arguments**: 3 reasons the hypothesis might be true.
3. **Falsification conditions**: "If X is true, this hypothesis is wrong."
4. **Confidence level**: 1-10 scale.

### Application Rules
1. Day 1 Answer is a testable hypothesis, NOT a conclusion.
2. Must define falsification conditions — an unfalsifiable hypothesis is not useful.
3. Update hypothesis as evidence accumulates (version log).
4. Combine with Issue Tree for structured decomposition.

## Three Horizons

### Structure
| Horizon | Time | Focus | KPIs | Resource Allocation |
|---------|------|-------|------|-------------------|
| H1 | 0-12 months | Core business | Revenue, profit, share | ~70% |
| H2 | 1-3 years | Growing businesses | Growth rate, CAC, LTV | ~20% |
| H3 | 3-10 years | Future seeds | Learning speed, options | ~10% |

### Application Rules
1. All three horizons must be actively managed simultaneously.
2. H3 investments should be small, numerous, and experimental.
3. H2 businesses need different governance than H1 (startup metrics, not P&L).
4. Review resource allocation quarterly — bias toward H1 is the most common failure.
