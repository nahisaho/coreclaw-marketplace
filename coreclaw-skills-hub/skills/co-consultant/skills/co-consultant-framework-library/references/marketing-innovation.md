# Marketing & Innovation Frameworks — Detailed Reference

Read this reference when applying marketing, customer analysis, or innovation frameworks.

## STP Analysis (Detailed)

### Structure
1. **Segmentation**: Divide market by demographics, psychographics, behavior, geography.
2. **Targeting**: Evaluate segment attractiveness (size, growth, competition, accessibility).
3. **Positioning**: Define unique value proposition for target segment.

### Application Rules
1. Segments must be MECE (no customer belongs to two segments).
2. Each segment must be measurable, accessible, substantial, and actionable.
3. Positioning statement format: "For [target], [brand] is the [category] that [benefit] because [reason]."

## Business Model Canvas (Detailed)

### 9 Building Blocks
| Block | Key Question |
|-------|-------------|
| Customer Segments | Who are we creating value for? |
| Value Propositions | What value do we deliver? |
| Channels | How do we reach customers? |
| Customer Relationships | How do we maintain relationships? |
| Revenue Streams | How do we earn money? |
| Key Resources | What assets are essential? |
| Key Activities | What activities are essential? |
| Key Partnerships | Who are our key partners? |
| Cost Structure | What are the major costs? |

### Application Rules
1. Start with Customer Segments and Value Propositions.
2. Ensure consistency: channels must reach the defined segments.
3. Revenue streams must align with value propositions.
4. Test: remove each block — does the model still work?

## TAM / SAM / SOM (Detailed)

### Definitions
| Level | Definition | Method |
|-------|-----------|--------|
| **TAM** (Total Addressable Market) | Total demand if 100% share | Top-down: industry reports |
| **SAM** (Serviceable Addressable Market) | Reachable with current model | TAM × geographic/segment filter |
| **SOM** (Serviceable Obtainable Market) | Realistically capturable | SAM × market share estimate |

### Application Rules
1. Always cite sources for TAM figures.
2. SAM filter criteria must be explicit and defensible.
3. SOM should reflect realistic Year 1-3 capture rate.
4. Include CAGR for growth projections.

## Design Thinking (Detailed)

### 5 Phases
| Phase | Goal | Key Activities | Output |
|-------|------|---------------|--------|
| **Empathize** | Understand users | Interviews, observation, immersion | User insights |
| **Define** | Frame the problem | Affinity mapping, POV statement | Problem statement |
| **Ideate** | Generate solutions | Brainstorming, SCAMPER, How Might We | Solution concepts |
| **Prototype** | Make it tangible | Low-fidelity prototypes, mockups | Testable prototypes |
| **Test** | Validate with users | User testing, feedback collection | Validated learnings |

### Application Rules
1. Phases are iterative, not linear. Return to Empathize after Test.
2. "How Might We" format: "How might we [action] for [user] so that [outcome]?"
3. Prototype at the lowest fidelity that tests the hypothesis.

## Lean Canvas (Detailed)

### Structure (vs BMC differences)
| Lean Canvas | BMC Equivalent | Why Different |
|-------------|---------------|--------------|
| Problem | — | Startups must validate problem first |
| Solution | Value Proposition | Solution follows problem |
| Key Metrics | — | Focus on actionable metrics |
| Unfair Advantage | Key Resources | What can't be copied |
| (same) | Channels, Revenue, Cost, Segments, Relationships | |

### Application Rules
1. Fill Problem and Customer Segments first.
2. "Unfair Advantage" should be defensible (not just "good team").
3. Key Metrics: choose 1 North Star metric per growth stage.
