# Executive Briefing

Options-based pyramid-structured decision packages.

## Use This Skill When

- Producing executive decision packages.
- Creating board-level transformation updates.
- Summarizing initiative progress for leadership.

## Workflow

1. Synthesize findings from prior phases.
2. Structure with pyramid principle (conclusion → arguments → evidence).
3. Present 2-3 options with explicit trade-offs.
4. State recommended option with rationale.
5. Include risks, mitigations, and next steps.
6. Save to files.

## Deliverables

- `report.md`: executive briefing.
- Reuse `assets/exec-briefing-template.md` when producing standardized briefings.

## Quality Gates

- [ ] 2-3 options presented (not a single recommendation).
- [ ] Trade-offs explicit for each option.
- [ ] Pyramid structure: conclusion → arguments → evidence.
- [ ] Risks and mitigations included.
- [ ] Document marked "DRAFT — CONFIDENTIAL".

If any gate fails: identify the issue, fix, and re-validate.

## Gotchas

- 「推奨オプション1つだけ」は意思決定支援にならない。必ず2-3の選択肢を提示すること
- エグゼクティブサマリーは全セクション完成後に書くこと
- 意思決定者の時間は限られている。1ページ（A4）で核心を伝えられるようにすること

## Validation Loop

1. ブリーフィングを生成
2. チェック: 2-3オプション、トレードオフ、ピラミッド構造、リスク
3. 不合格なら修正
4. ユーザー承認後のみ完了 ⏸️
