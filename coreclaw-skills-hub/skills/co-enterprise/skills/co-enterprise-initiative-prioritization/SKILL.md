# Initiative Prioritization

Weighted scoring by impact × feasibility × risk × strategic fit.

## Use This Skill When

- Scoring and ranking transformation initiatives.
- Building a prioritized initiative portfolio.
- Allocating resources across competing programs.

## Workflow

1. List candidate initiatives.
2. Define scoring criteria and weights (agree with stakeholders before scoring).
3. Score each initiative (1-5) per criterion.
4. Calculate weighted scores and rank.
5. Visualize: prioritization matrix (Impact vs Feasibility).
6. Save to files.

## Deliverables

- `results/initiative-prioritization.md`: scored portfolio.
- Reuse `assets/prioritization-matrix-template.md` when producing standardized scorecards.

## Quality Gates

- [ ] Scoring criteria and weights agreed before evaluation.
- [ ] All initiatives scored consistently.
- [ ] Top initiatives align with strategic objectives.
- [ ] Trade-offs between top candidates documented.

If any gate fails: identify the issue, fix, and re-validate.

## Gotchas

- 評価基準のウェイトは事前に合意すること。スコアリング後に変更すると政治的対立を招く
- 「全部優先度高」は優先順位付けではない。強制ランキングで差をつけること
- Feasibility は「技術的実現可能性」だけでなく「組織的受容性」も含めること

## Validation Loop

1. スコアリング結果を生成
2. チェック: ウェイト事前合意、一貫性、戦略整合性
3. 不合格なら修正
4. ユーザー承認後のみ完了 ⏸️
