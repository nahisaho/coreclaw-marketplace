# フレームワークライブラリ

> 53のコンサルティングフレームワーク定義と選択支援を提供します。

## フレームワーク一覧

### 1. 戦略分析（10定義）

#### SWOT分析
- **目的**: 内部の強み・弱みと外部の機会・脅威を整理
- **構成要素**: Strengths, Weaknesses, Opportunities, Threats
- **用途**: 戦略立案の基礎、現状把握

#### 3C分析
- **目的**: 市場環境を3つの視点で分析
- **構成要素**: Customer（市場・顧客）, Competitor（競合）, Company（自社）
- **用途**: マーケティング戦略、競合対策

#### PEST分析
- **目的**: マクロ環境を4つの視点で分析
- **構成要素**: Political（政治）, Economic（経済）, Social（社会）, Technological（技術）
- **用途**: 市場参入判断、事業環境分析

#### 5 Forces分析
- **目的**: 業界の競争環境を5つの力で分析
- **構成要素**: 新規参入の脅威, 代替品の脅威, 買い手の交渉力, 売り手の交渉力, 業界内の競争
- **用途**: 業界分析、参入判断

#### BCGマトリクス
- **目的**: 事業ポートフォリオを可視化
- **構成要素**: 花形（Star）, 金のなる木（Cash Cow）, 問題児（Question Mark）, 負け犬（Dog）
- **用途**: 事業ポートフォリオ管理、資源配分

#### VRIO分析
- **目的**: 経営資源の競争優位性を評価
- **構成要素**: Value（価値）, Rarity（希少性）, Imitability（模倣困難性）, Organization（組織）
- **用途**: 競争優位の源泉特定

#### GEマトリクス
- **目的**: 事業の魅力度と競争力を評価
- **構成要素**: 市場の魅力度（高/中/低）× 競争力（高/中/低）
- **用途**: 多角化企業の事業評価

#### アンゾフマトリクス
- **目的**: 成長戦略の方向性を検討
- **構成要素**: 市場浸透, 市場開拓, 製品開発, 多角化
- **用途**: 成長戦略立案

#### ブルーオーシャン戦略
- **目的**: 競争のない市場空間を創造
- **構成要素**: 戦略キャンバス, 4つのアクション（排除・削減・増加・創造）
- **用途**: 差別化戦略、新市場創造

#### バリューチェーン分析
- **目的**: 企業活動を分解して価値創造を分析
- **構成要素**: 主活動（購買→製造→出荷→販売→サービス）, 支援活動
- **用途**: コスト削減、競争優位の特定

### 2. 問題解決（7定義）

#### MECE
- **目的**: 漏れなくダブりなく整理
- **構成要素**: Mutually Exclusive（相互排他）, Collectively Exhaustive（全体網羅）
- **用途**: あらゆる分析の基本原則

#### ロジックツリー
- **目的**: 問題を論理的に分解
- **構成要素**: What Tree（要素分解）, Why Tree（原因分析）, How Tree（解決策）
- **用途**: 問題の構造化、原因分析

#### イシューツリー
- **目的**: 問題をMECEに分解して優先順位付け
- **構成要素**: 中心イシュー → サブイシュー → 仮説
- **用途**: 問題の特定、仮説検証計画

#### 5 Whys分析
- **目的**: 根本原因を特定
- **構成要素**: 5回の「なぜ？」による掘り下げ
- **用途**: 根本原因分析、問題の深掘り

#### フィッシュボーン（石川ダイアグラム）
- **目的**: 問題の原因を網羅的に特定
- **構成要素**: 結果 ← 大骨（カテゴリ）← 中骨 ← 小骨（原因）
- **用途**: 品質管理、原因分析

#### 空雨傘
- **目的**: 事実→解釈→行動を構造化
- **構成要素**: 空（事実）, 雨（解釈）, 傘（行動）
- **用途**: 意思決定、日常的な問題解決

#### As-Is/To-Be分析
- **目的**: 現状と理想を比較してギャップを特定
- **構成要素**: As-Is（現状）, To-Be（理想）, Gap（差分）
- **用途**: 改善計画、変革管理

### 3. 思考整理（3定義）

#### ピラミッド構造（Minto Pyramid）
- **目的**: 結論先行で論理的にコミュニケーション
- **構成要素**: 結論 → 根拠（3つ程度）→ データ
- **用途**: レポート構成、プレゼンテーション

#### So What / Why So
- **目的**: 論理の飛躍を防止
- **構成要素**: So What?（だから何？）, Why So?（なぜそう言える？）
- **用途**: 論理チェック、分析の深化

#### PREP法
- **目的**: 説得力のある説明構造
- **構成要素**: Point（結論）, Reason（理由）, Example（具体例）, Point（結論再提示）
- **用途**: プレゼンテーション、説明
