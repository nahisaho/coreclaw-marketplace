# Framework Library (MCK)

9 McKinsey-specific framework definitions.

## Use This Skill When

- Looking up a MCK-specific framework definition.
- Comparing MCK frameworks for a business question.
- Selecting the best framework from MCK candidates.

## Framework Index

| Framework | Primary Trigger | Category |
|-----------|----------------|----------|
| McKinsey 7S | 7S | organization |
| Hypothesis-Driven Approach | hypothesis | problem-solving |
| Issue Tree | issue decomposition | problem-solving |
| Pyramid Principle (Minto) | pyramid | communication |
| Three Horizons | growth horizons | growth |
| Granularity of Growth | granular | growth |
| OVA (Overhead Value Analysis) | overhead | cost |
| Profit Pool Analysis | profit pool | strategy |
| SCP Analysis | SCP | strategy |

## Recommended Workflow Patterns


### Organization Transformation
`McKinsey 7S` → `Issue Tree` → `OVA` → `Pyramid Principle (Minto)`

### Growth Strategy
`Three Horizons` → `Granularity of Growth` → `Profit Pool Analysis`

### Problem Solving
`Hypothesis-Driven Approach` → `Issue Tree` → `SCP Analysis` → `Pyramid Principle (Minto)`


## Selection Principles

1. Choose the MCK framework that directly answers the core question.
2. Prefer simpler frameworks when data is limited.
3. Maximum 3 frameworks per analysis.
4. MCK frameworks take priority over generic frameworks when applicable.
5. Read detailed definitions in `references/frameworks.md` when applying a specific framework.

## Gotchas

- MCKフレームワークは「考える道具」であって「答え」ではない。フレームワークを埋めることが目的になってはならない
- 汎用フレームワーク（SWOT, 3C等）とMCKフレームワークは併用可能だが、MCK固有の視点を主軸にすること
- フレームワーク間の推奨ワークフロー順序を無視して適用すると、前提となるデータが不足する

## Validation Loop

1. フレームワーク情報を提供
2. チェック: 定義が正確か、選択理由が目的と合致しているか
3. 不合格なら修正
