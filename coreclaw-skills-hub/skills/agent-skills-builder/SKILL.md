# Agent Skills Builder

CoreClaw向けのSkillを、対話ベースで要件発見しながら作るビルダーです。

## 目的

- Skill作成の標準化
- ユーザーニーズの深掘り
- 提案による要件の具体化
- `skill.json` の整合性確保
- ドキュメント品質の担保
- リリース運用ミスの削減

## 対話方針

- ユーザーへの質問を通じて、背景・目的・制約を引き出す
- 要件が曖昧な場合は、こちらから複数案を積極提案する
- 提案時は、メリット・デメリット・推奨理由を簡潔に示す
- 最終的に、ユーザーが選べる形で仕様を確定する

## 想定ワークフロー

1. 対話で課題と目的を明確化
2. 活用シーンと期待成果を整理
3. こちらから複数のSkill構成案を提案
4. 方針確定後に必須ファイル生成（`skill.json`, `main.py`, `README.md`, `SKILL.md`）
5. 必要に応じてサブスキル構成と `group.json` を生成
6. 命名・バージョン・entrypointの検証
7. リリースタグの提案

## 入力

- Skill名
- 対象グループ（例: `scientist`, `enterprise`）
- Skillの目的と利用者
- 必要な入出力形式
- バージョン方針（通常は `v0.1.0` 開始）
- 運用条件（納期、品質要件、依存関係）

## 出力

- 作成されたSkillパッケージ
- 提案比較（採用案/代替案）
- バリデーション結果
- リリース手順案

## 文字数制約

- Skill本体の記述は1500文字以内
- サブスキルの記述は2000文字以内
- 制約超過時は要約して再生成する

## 品質基準

- 必須フィールドを満たす `skill.json`
- `entrypoint` と実ファイル一致
- 説明文は明確で簡潔
- リポジトリ規約（命名、構成）に準拠
- 対話ログに基づいた要件反映が確認できる
