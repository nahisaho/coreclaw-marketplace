# Purpose Discovery (ACN)

Discover the client's true objective through structured dialogue.

## Use This Skill When

- Starting a new Accenture-style consulting engagement.
- The request is ambiguous or surface-level.
- Defining scope, constraints, and success criteria.

## Workflow

1. Decompose request into 6 elements (PURPOSE, TARGET, SCOPE, TIMELINE, CONSTRAINTS, DELIVERABLES).
2. Present for user approval ⏸️.
3. One-question-at-a-time dialogue (minimum 3 rounds).
4. 5 Whys analysis for abstract answers.
5. JTBD analysis (Functional, Emotional, Social jobs).
6. Define true objective → user approval ⏸️.
7. Generate research plan with bilingual search queries.
8. Recommend ACN-specific frameworks based on the objective.

## Deliverables

- `results/purpose-definition.md`: 6-element breakdown + true objective.
- `results/research-plan.md`: search queries, recommended ACN frameworks.

## Quality Gates

- [ ] All 6 elements addressed.
- [ ] Minimum 3 dialogue rounds completed.
- [ ] True objective is specific, actionable, and user-approved.
- [ ] Recommended ACN frameworks are justified.

## Gotchas

- 複数の質問を一度に投げてはならない。必ず1問1答形式で進める
- ユーザーの最初の依頼が「真の目的」であることは稀。表層的な依頼の裏にある本質を探ること
- ACNフレームワークの推薦は目的確定後に行う。目的が曖昧な段階でフレームワークを決めない

## Validation Loop

1. 6要素分解と真の目的を生成
2. チェック: PURPOSE が意思決定に直結するか、SCOPE が検証可能か
3. 不合格なら対話を追加
4. ユーザー承認後のみ完了
