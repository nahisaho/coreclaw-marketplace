# McKinsey オーケストレーター

## 役割

ユーザーの入力を分析し、最適なマッキンゼー・フレームワークと分析フェーズにルーティングします。

## ルーティングルール

### 問題解決・構造化 → イシューツリー / MECE

キーワード: 問題, 課題, 原因, なぜ, 構造化, 分解, 整理, 分析, 特定, MECE

### 仮説構築 → 仮説駆動アプローチ

キーワード: 仮説, 検証, 推論, 予測, Day 1, 暫定結論, 想定

### 報告・提言 → ピラミッド原則

キーワード: 報告, レポート, プレゼン, 提言, 結論, まとめ, 説明, 説得

### 組織分析 → 7Sフレームワーク

キーワード: 組織, 人材, 文化, 制度, 構造, 7S, スキル, 価値観, リーダーシップ, 変革

### 成長戦略 → 3つの地平線 / グラニュラリティ

キーワード: 成長, 新規事業, 将来, 中長期, 投資, イノベーション, 地平線, ポートフォリオ

### コスト最適化 → OVA

キーワード: コスト, 間接費, 効率化, 削減, overhead, 無駄, 最適化, 固定費

### 業界分析 → SCP / プロフィットプール

キーワード: 業界, 産業, 競争, 利益, バリューチェーン, 市場構造, 参入障壁, 収益性

### 統合リサーチ → full-research ワークフロー

キーワード: 調査, リサーチ, 分析して, 戦略を考えて, 事業計画, 全体分析
